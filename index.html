Google Drive Setup — One Time
  1. Go to console.cloud.google.com and create a project named JD Pipeline.
  2. Enable the Google Drive API under APIs & Services > Library.
  3. Go to OAuth consent screen. User type: External. Add your Gmail as a test user.
  4. Go to Credentials > Create Credentials > OAuth client ID. Type: Web application. Under Authorized JavaScript origins add http://localhost:8080 and http://localhost.
  5. Copy the Client ID, paste it in the field above, and click Save Profile.
  6. Click Connect Google. Approve Drive access in the popup.
  7. On first sync, jd-pipeline.json is created in your Drive root.
Note: While your app is in test mode, only Gmail accounts you add as test users can authenticate. This is fine for personal use indefinitely.
OneDrive Setup — One Time
  1. Go to portal.azure.com and sign in with your Microsoft account.
  2. Search for App registrations and click New registration.
  3. Name it JD Pipeline. Under Supported account types select Personal Microsoft accounts only. Leave redirect URI blank for now. Click Register.
  4. Copy the Application (client) ID from the overview screen and paste it in the field above.
  5. In the left menu click Authentication. Under Platform configurations click Add a platform and select Single-page application. Set the redirect URI to: https://chris-trolitsch.github.io/jd-pipeline/. Click Configure.
  6. Still under Authentication, scroll to Advanced settings and enable Allow public client flows. Click Save.
  7. Click Save Profile in the dashboard, then click Connect OneDrive. A Microsoft popup will ask for sign-in and Files permission.
  8. On first sync, jd-pipeline.json is created in your OneDrive root.
Note: The redirect URI must exactly match your GitHub Pages URL including the trailing slash. MSAL handles the popup flow — no server-side code is required.
